# Cull Messaging Guide

Status: working direction, not final copy deck.

## Primary Message

Cull is a local-first reference engine for AI creativity, built so humans and agents can turn visual chaos into coherent projects, publications, and stories.

## Short Descriptions

Use these for compact product contexts:

- Agent-native archive for AI-generated creative work.
- A local-first reference engine for images, videos, prompts, and visual memory.
- Creative memory for agents and the people who work with them.
- Open-source software for turning unsorted AI work into collections, posts, books, and exhibitions.
- A private, local archive that agents can search, curate, and compose through MCP.

## Hero Lines

- From visual chaos to coherent work.
- Creative memory for agents.
- Your local reference engine for AI creativity.
- Cull turns piles of AI-generated images into projects, publications, and memories.
- A visual archive that agents can actually use.

## Supporting Copy

Cull organizes AI-generated images, videos, prompts, and references into a local archive that agents can search and operate through MCP. It helps transform messy folders into coherent collections, social posts, publications, books, exhibitions, and reusable creative memory.

## Agent-Native Message

Cull is designed for agent control from the ground up. Its MCP tools let agents search the archive, inspect metadata, build collections, apply selections, compose exports, and help turn raw creative material into finished outputs.

## Privacy Message

Cull is local-first and privacy-first. Your archive stays on your machine by default, without hidden data collection or forced dependency on a cloud subscription.

## Open Source Message

Cull is open source so the creative community can inspect, extend, and adapt the tool instead of trusting a black-box archive for their visual memory.

## European Trust Message

Cull is made in Europe and should meet European expectations for privacy, autonomy, transparency, and lawful data handling. Any cloud or API integration should be explicit, optional, and auditable.

## Use Case Messages

### Visual References

Cull can work like a local Pinterest for serious creative reference: save, search, cluster, and export visual references that agents can access through MCP.

### Social Publishing

Cull helps turn large batches of images into coherent Instagram or LinkedIn posts by selecting, ordering, annotating, and exporting collections.

### Publications

Cull supports the path from archive to publication: books, magazines, exhibitions, field guides, essays, and visual stories.

### Community

Cull can become infrastructure for an AI creativity community: a shared language for collecting, curating, and publishing the work that emerges from agentic creative practice.

## Message Pillars

- **Control:** your archive, on your machine, under your rules.
- **Memory:** save visual work in a way humans and agents can reuse.
- **Coherence:** transform chaotic output into structured creative projects.
- **Agency:** agents can operate the archive through MCP.
- **Openness:** transparent, extensible, community-driven software.

## Avoid

- "AI-powered productivity platform"
- "all-in-one content solution"
- "cloud-native media hub"
- "growth engine"
- "10x your workflow"
- claims about legal compliance before an audit exists
- claims that no data ever leaves the device if optional cloud API features are enabled
