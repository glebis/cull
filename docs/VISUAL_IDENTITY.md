# Cull Visual Identity Guide

Status: working direction, not final production guidelines.

## Direction

Cull's visual identity combines:

- Bauhaus geometry
- terminal systems
- modern editorial design
- art-book and exhibition graphics
- European privacy-tech restraint
- surreal creative memory imagery

The design should feel clear, modern, sleek, and culturally specific. It should not feel like a generic AI SaaS landing page.

## Logo

Primary wordmark: `Cull`

The current direction uses a heavy geometric wordmark/icon system built around a large `C` and two vertical bars. It should feel blunt, memorable, and constructed rather than friendly or ornamental.

Alternate product title: `cull.company`

Use `cull.company` as a backup title or small lockup when the domain itself is part of the message.

## Icon Style

The active app icon direction is flat and minimal, not glossy.

Current icon variants:

- Primary Mono: off-white / black
- Signal Red: red / off-white
- Bauhaus Blue: blue / off-white
- Dark Mono: black / off-white
- Archive Yellow: yellow / black

The Tahoe-compatible icon files should keep the proper rounded-square mask behavior, but the brand should avoid forced Liquid Glass gradients unless we intentionally create a separate platform-native variant.

## Color

Brand palette:

| Name | Hex | Use |
| --- | --- | --- |
| Off-white | `#F6F6F2` | paper, backgrounds, editorial surfaces |
| Black | `#0A0A0A` | wordmark, terminal, high-contrast UI |
| Signal Red | `#E10000` | primary brand accent, calls to attention |
| Bauhaus Blue | `#005EFF` | system accent, selected states, graphic bars |
| Archive Yellow | `#FFD400` | stamps, labels, secondary emphasis |

App UI tokens remain governed by `docs/design-system.md` and `src/app.css`.

## Typography

Use two typographic poles:

- A refined monospace for system, terminal, metadata, captions, and product UI language.
- EB Garamond or a similar editorial serif for large quotes and reflective brand moments.

Monospace should feel fancier than raw terminal defaults when used for brand pages. The look should be technical but not developer-only.

Avoid using many type families. The contrast should be monospace system precision against serif editorial memory.

## Layout

Layout should use:

- asymmetry
- large blank space
- wide bold typographic inserts
- hard grid logic
- strong black/off-white contrast
- full-bleed image blocks without borders
- editorial pacing across several screens

Avoid:

- clinical one-to-one feature grids
- rounded SaaS cards everywhere
- decorative gradient blobs
- generic split hero sections
- too many framed mockups

## Imagery

The recurring visual metaphor is transformation:

messy visual material -> coherent output.

Show the transformation into:

- exhibitions
- books
- magazines
- publications
- Instagram posts
- LinkedIn posts
- visual stories
- memory collections

Large image-only blocks are part of the system. Not every image needs a border or explanation.

## Art Direction With Yulia Katan Works

Yulia Katan's drawings and object images can be used because written permission has been confirmed by the project owner.

Use them as real art material, not background decoration. They can carry the emotional part of the brand: surreal memory, transformation, hybrid objects, and visual imagination.

Keep usage respectful:

- preserve attribution in asset notes
- avoid destructive cropping that misrepresents the work
- do not train or generate derivative art from the works unless permission explicitly covers that
- keep source/provenance notes near any imported files

## Website Direction

The website should start with a large, spare hero. It should not feel like a marketing template.

Promising directions:

- blank, confident first screen with one strong transformation image
- large EB Garamond user quote blocks
- terminal/MCP proof sections
- image-only art/editorial sections
- output variety sections showing social posts, publications, exhibitions, and books
- privacy/open-source sections expressed visually, not as dense policy text
