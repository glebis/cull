# Release Readiness Execution Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Execute the release-readiness audit spec (`docs/superpowers/specs/2026-06-09-release-readiness-audit-design.md`) end to end: Stage 1 multi-agent audit → decision gate → Stage 2 bd plan → TDD execution of Tracks A/B/C → built, reinstalled, running app — with a codex audit/negotiate/commit/push gate after every stage.

**Architecture:** Two Workflow-tool scripts produce the audit and the plan; bd issues drive track execution; a repeating codex gate closes each stage. Track A/B/C issue *content* is generated by Stage 2 (by design) — this plan fixes the loop contract, the issue template, and every command, so generated issues are executable without further decisions.

**Tech Stack:** Claude Code Workflow tool, codex CLI (read-only sandbox), bd (beads), cargo/vitest, Tauri 2 bundler, codesign/spctl/stapler.

**Stage gate (applies after Tasks 2, 5, 7, 8, 9, 11 — referenced as "CODEX GATE"):**

1. Run codex on the stage's artifacts/diff (exact command given per task).
2. If verdict is REWORK or APPROVE WITH CHANGES: triage requirements, push back on out-of-scope items via `echo "<negotiation>" | codex exec --skip-git-repo-check resume --last 2>/dev/null`, loop until APPROVE / APPROVE WITH CHANGES with all conditions either implemented or explicitly deferred-with-rationale.
3. Implement accepted changes (TDD for code), append outcome to the spec's Review log.
4. Commit with message `<type>: <summary> (codex gate: stage N)` and push: `git pull --rebase && git push` (use `CULL_PREFLIGHT_SKIP_E2E=1 git push` when the diff has no UI/browser-covered areas; frontend-only branches may need `--no-verify` per repo memory).
5. If codex CLI hangs >7 min or errors, fall back to `feature-dev:code-reviewer` agent and mark the gate "codex-substitute" in the commit message.

```javascript
export const meta = {

const INVENTORY = { type: 'object', required: ['items'], properties: { items: { type: 'array', items: {

  }}}}}

const FINDINGS = { type: 'object', required: ['findings'], properties: { findings: { type: 'array', items: {
  type: 'object',
  required: ['id','lens','title','severity','evidence','proposed_fix','est_effort'],
  properties: {
    id:{type:'string'}, lens:{type:'string'}, title:{type:'string'},
    severity:{enum:['release-blocker','pre-launch','post-launch']},
    evidence:{type:'array', items:{type:'string'}}, proposed_fix:{type:'string'},
    est_effort:{enum:['S','M','L']}, runtime_verified:{type:'boolean'},
  }}}}}

const VERDICT = { type:'object', required:['refuted','reason'], properties:{ refuted:{type:'boolean'}, reason:{type:'string'} } }

phase('Inventory')
const [ui, mcp] = await parallel([
  () => agent(`${FORBIDDEN}\nBuild a complete inventory of Cull's user-facing UI features: every route under src/routes, every panel/modal/component in src/lib, every command-palette command, every keyboard shortcut, import/export/publish flows.   () => agent(`${FORBIDDEN}\nBuild a complete inventory of Cull's agent surface: every MCP tool registered in src-tauri (search for the MCP server registration), grouped by capability (browse/curate/collections/quality/export/publish/tokens/jobs). Same schema. test_coverage from Rust tests covering the handler.`, {label:'inventory:mcp', phase:'Inventory', schema: INVENTORY, agentType:'Explore'}),
])
let items = [...(ui?.items ?? []), ...(mcp?.items ?? [])]
log(`inventory: ${items.length} items`)

const gap = await agent(`${FORBIDDEN}\nCompleteness gate. Here is a feature inventory of Cull: ${JSON.stringify(items.map(i=>({id:i.id,name:i.name,surface:i.surface})))}.\nCross-check against: src/routes/* files, command palette registry, keyboard shortcut map, MCP tool registration list in src-tauri. Return ONLY missing items in the same schema (empty array if complete).`, {label:'inventory:gate', phase:'Inventory', schema: INVENTORY})
items = [...items, ...(gap?.items ?? [])]
log(`inventory complete: ${items.length} items after gap fill`)

phase('Lenses')
const LENSES = [
  {key:'code-quality', prompt:'Audit Rust backend (unwrap()/panic paths, Mutex poisoning, migration safety, error propagation in Tauri commands in src-tauri/src/commands and db_core) and Svelte 5 frontend (the $effect-reads-state-it-mutates trap class, store misuse, dead components in src/lib) and IPC error handling.'},
  {key:'security', prompt:'Audit security & privacy: secrets or personal paths in working tree AND git history (git log -p sampling, .env patterns), path traversal in import/export, MCP token lifecycle (create/rotate/revoke in commands), Tauri capabilities/CSP in src-tauri/tauri.conf.json and capabilities/, ALL outbound network calls (model downloads, updater endpoint), audit-log coverage.'},
  {key:'hygiene', prompt:'Audit release hygiene: license metadata alignment across LICENSE.md, NOTICE, package.json, src-tauri/Cargo.toml, README, About dialog INCLUDING uncommitted drift (git status); run npm run audit:licenses and npm run audit:supply-chain and report results; lockfiles committed; repo-going-public cleanup (.beads leakage, personal paths, docs that should not ship); SECURITY.md and release-notes presence; README adequacy for a stranger; signing/notarization config in tauri.conf.json; upgrade path — confirm migration_safety_tests cover current schema version.'},
  {key:'ux', prompt:'Audit UX as a stranger in their first 10 minutes. Drive the REAL UI: dev server at localhost:1420, agent-browser via CDP on port 9222 (read AGENTS.md "agent-browser + Svelte 5 patterns" first; use tab new, CSS-selector clicks, native value setter for inputs). Cover: empty states, first-run import flow, error states, keyboard discoverability, Tokyo Night token consistency, terminology coherence. Accessibility basics: contrast of token palette, focus order import→grid→loupe→export, aria-labels on interactive controls. Mark each finding runtime_verified true/false (false = code-traced only).'},
  {key:'performance', prompt:'Audit performance & scale against these 10k-image pass/fail thresholds: cold start to interactive grid < 3s; thumbnail load p95 < 200ms with no unbounded memory growth; smart-collection/NL query p95 < 500ms; CLIP find-similar < 2s; loupe open < 1s full-res (cached thumb < 300ms); resident memory < 1.5GB. State measured-vs-threshold per finding.'},
]
const lensResults = await pipeline(
  LENSES,
  l => agent(`${FORBIDDEN}\nYou are the ${l.key} audit lens for Cull (read AGENTS.md first; spec context: ${SPEC}). ${l.prompt}\nSeverity: release-blocker = a stranger or the public repo is harmed; pre-launch = fix before soft launch if time allows; post-launch = file and move on.`, {label:`lens:${l.key}`, phase:'Lenses', schema: FINDINGS}),

    agent(`Adversarially REFUTE this release-blocker finding about Cull. Default refuted=true unless the evidence stands up to verification. Finding: ${JSON.stringify(f)}. Check the cited files/lines yourself.`, {label:`verify:${f.id}`, phase:'Verify', schema: VERDICT})
      .then(v => ({...f, severity: v?.refuted ? 'pre-launch' : f.severity, refutation: v?.refuted ? v.reason : null}))
  )).then(verified => {
    const rest = (res?.findings ?? []).filter(f => f.severity !== 'release-blocker')
    return [...verified.filter(Boolean), ...rest]
  })
)
const findings = lensResults.filter(Boolean).flat()
log(`${findings.length} findings, ${findings.filter(f=>f.severity==='release-blocker').length} verified blockers`)

phase('Identity')
const IDENTITIES = ['fast culling tool — keyboard-driven triage of large AI-generation batches',
  'AI art library — browse, CLIP semantic search, collections, generation metadata',
  'agent-native image tool — the MCP surface as headline differentiator']
const ADV = {type:'object', required:['identity','argument','strongest_features','weaknesses'], properties:{identity:{type:'string'}, argument:{type:'string'}, strongest_features:{type:'array',items:{type:'string'}}, weaknesses:{type:'array',items:{type:'string'}}}}
const advocates = await parallel(IDENTITIES.map(idn => () =>
  agent(`${FORBIDDEN}\nArgue at FULL strength that Cull's release identity should be: "${idn}". Cite only this inventory and these findings as evidence:\nINVENTORY: ${JSON.stringify(items)}\nFINDINGS (titles+severity): ${JSON.stringify(findings.map(f=>({t:f.title,s:f.severity,lens:f.lens})))}`, {label:`advocate`, phase:'Identity', schema: ADV})))

const judge = await agent(`Judge these three identity arguments for Cull's release (5-day budget, soft launch to friends). Score each 1-10 on differentiation, code maturity (from findings), soft-launch audience, budget fit. Recommend ONE. Preserve the losers' strongest claims as keep_anyway.\n${JSON.stringify(advocates.filter(Boolean))}`, {label:'judge', phase:'Identity', schema: JUDGE})
log(`identity: ${judge.recommended_identity}`)

phase('Triage')
const TRIAGE = {type:'object', required:['verdicts'], properties:{verdicts:{type:'array', items:{type:'object', required:['item_id','verdict','rationale'], properties:{item_id:{type:'string'}, verdict:{enum:['CORE','KEEP','DEMOTE','PLUGIN','CUT']}, rationale:{type:'string'}, est_effort:{enum:['S','M','L']}}}}}}
const SACRED = 'SACRED (may not be CUT or PLUGIN): the MCP server itself, CLIP semantic search (embeddings/find-similar/NL search), smart collections.'
let triage = await agent(`${FORBIDDEN}\nTriage EVERY inventory item against the recommended identity "${judge.recommended_identity}" (rationale: ${judge.rationale}). ${SACRED}\nVerdicts: CORE (bullish, gets polish), KEEP (ship as-is), DEMOTE (hide behind settings), PLUGIN (extraction candidate + effort), CUT (remove). const sacredViolation = (triage?.verdicts ?? []).filter(v => /mcp.server|clip|semantic|find.similar|smart.collection|nl.search/i.test(v.item_id) && ['CUT','PLUGIN'].includes(v.verdict))
if (sacredViolation.length) {
  triage = await agent(`Your triage violated the sacred constraint for: ${JSON.stringify(sacredViolation)}. ${SACRED} Re-issue the FULL verdict list with those corrected to CORE or KEEP.\nOriginal: ${JSON.stringify(triage.verdicts)}`, {label:'triage:fix', phase:'Triage', schema: TRIAGE})
}

phase('Plugin')
const pluginSpec = await agent(`${FORBIDDEN}\nDesign Cull's plugin mechanism + store, working backwards from these PLUGIN-verdict features: ${JSON.stringify((triage?.verdicts ?? []).filter(v=>v.verdict==='PLUGIN'))} (full inventory items: ${JSON.stringify(items.filter(i => (triage?.verdicts ?? []).some(v=>v.item_id===i.id && v.verdict==='PLUGIN')))}).\nRequired sections (read spec ${SPEC} Phase 4 for exact requirements): mechanism choice (frontend JS modules vs MCP tool packs vs external process, vs hybrid) with rationale against Tauri 2 constraints and the 5-day budget; manifest format {id,name,version,description,entry,permissions,minAppVersion,checksum,repo}; registry v1 = single registry.json in a public cull-plugins GitHub repo over HTTPS with checksum verify, plus migration path to signed index/API; security model consistent with MCP token/audit-log posture; honest sizing in hours for runtime, registry fetch + install UX, one proof plugin — committed budget ~12h, if sizing >16h cut plugin features not track time; Day-4 fallback valve and its machine-checkable acceptance (spec Phase 4). Return the full spec as markdown text.`, {label:'plugin-spec', phase:'Plugin'})

phase('Synthesis')
await agent(`Write Stage 1 audit outputs for Cull. Create directory docs/release-audit-2026-06-09/ and write:\n1. inventory.json — ${JSON.stringify({count: items.length})} items provided below.\n2. findings.json — findings below.\n3. report.md — verified findings grouped by severity (blockers first, each with evidence and refutation notes), identity panel summary (all three arguments + judge scores + recommendation), full triage table, plugin spec (verbatim below), and a "completeness" section noting any lens that narrowed scope or finding not runtime-verified.\n4. decision-sheet.md — machine-parseable markdown table, EXACT columns: | item_id | type | audit_recommendation | user_decision | override_reason | evidence_ids |. One row per taste call: row id=identity (type=identity, recommendation=${JSON.stringify(judge?.recommended_identity)}), one row per non-KEEP triage verdict (type=triage), one row id=plugin-scope (type=plugin-scope). Pre-fill user_decision with the recommendation. item_id values must be unique. evidence_ids reference findings.json/inventory ids; may be empty only where the source finding is marked runtime_verified=false.\n\nDATA:\nINVENTORY=${JSON.stringify(items)}\nFINDINGS=${JSON.stringify(findings)}\nJUDGE=${JSON.stringify(judge)}\nADVOCATES=${JSON.stringify(advocates.filter(Boolean).map(a=>({identity:a.identity, argument:a.argument})))}\nTRIAGE=${JSON.stringify(triage?.verdicts)}\nPLUGIN_SPEC:\n${pluginSpec}`, {label:'synthesis', phase:'Synthesis'})

const critic = await agent(`Completeness critic. Read docs/release-audit-2026-06-09/report.md and decision-sheet.md. Check: every inventory item has a triage verdict (inventory had ${items.length} items); every release-blocker has a verification note; decision-sheet item_ids unique, columns exact, every non-KEEP verdict present; no lens silently absent. Fix the files in place for mechanical gaps; return a list of substantive gaps you could not fix.`, {label:'critic', phase:'Synthesis'})
return { items: items.length, findings: findings.length, blockers: findings.filter(f=>f.severity==='release-blocker').length, identity: judge?.recommended_identity, critic_gaps: critic }
```

### Task 6: Execute Track A (TDD, per-issue loop)

Issue content comes from bd (created in Task 4). The loop below is the fixed contract for **every** issue in Tracks A, B, C; execute issues in `day` order, respecting `depends_on`.

- [ ] **Step 1: Claim** — `npm run bd -- update <id> --status in_progress`
- [ ] **Step 2: Write the failing test named in the issue's "Test plan" line** (Rust: `src-tauri/src/.../tests`; frontend: `src/lib/*.test.ts`; docs-only issues: skip to Step 5 and use the issue's verification command instead).
- [ ] **Step 3: Run it, verify it FAILS** — Rust: `cd src-tauri && cargo test --lib <test_name>`; frontend: `npx vitest run <file> -t "<name>"`. Expected: FAIL for the reason the issue describes.
- [ ] **Step 4: Implement minimally; run the test, verify PASS; then the full suite for the touched side** — Rust: `cd src-tauri && cargo fmt && cargo test --lib`; frontend: `npm run check && npm test`.
- [ ] **Step 5: Commit + close** — `git add -A && git commit -m "<type>: <issue title> (bd <id>)" && npm run bd -- close <id>`
- [ ] **Step 6: After the LAST Track A issue:** `npm run preflight:full` — expected exit 0.
