# Cull Site Landing Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the `/site` Vercel landing page for `cull.company` with confirmed opt-in email signup.

**Architecture:** `/site` is a separate Vite static app with Vercel Node Functions under `/site/api`. Signup logic is split into pure modules for validation, hashing, state transitions, Blob persistence, and Resend delivery so the critical behavior can be tested without network calls.

**Tech Stack:** Vite, TypeScript, Vitest, Vercel Node Functions, `@vercel/blob`, Resend.

---

## File Structure

- Create `site/package.json`: independent app scripts and dependencies.
- Create `site/package-lock.json`: lockfile generated by `npm install`.
- Create `site/index.html`: static page shell.
- Create `site/src/main.ts`: signup form behavior and query-state rendering.
- Create `site/src/styles.css`: Tokyo Night-inspired landing page styling.
- Create `site/src/signup/types.ts`: signup domain types.
- Create `site/src/signup/email.ts`: email normalization and validation.
- Create `site/src/signup/crypto.ts`: token generation and HMAC hashing.
- Create `site/src/signup/time.ts`: expiry helpers.
- Create `site/src/signup/rate-limit.ts`: low-volume Blob-backed rate-limit helpers.
- Create `site/src/signup/store.ts`: Blob persistence interface and implementation.
- Create `site/src/signup/resend.ts`: confirmation email boundary.
- Create `site/src/signup/service.ts`: subscribe/confirm orchestration.
- Create `site/api/subscribe.ts`: Vercel Function for signup submissions.
- Create `site/api/confirm.ts`: Vercel Function for confirmation links.
- Create `site/src/signup/*.test.ts`: focused unit tests.
- Create `site/vercel.json`: Vercel build and ignored-build configuration.
- Create `site/README.md`: deployment/env notes.
